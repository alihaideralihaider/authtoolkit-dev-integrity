# AuthToolkit Brand System

AuthToolkit is an infrastructure brand for authentication, recovery, revenue, API execution, agent operations, webhooks, events, and operational workflows.

This brand system defines the reusable visual and messaging language for future AuthToolkit landing pages, dashboards, documentation pages, social graphics, product UIs, onboarding flows, and internal marketing assets.

## Brand Position

AuthToolkit should feel like operational infrastructure for teams that need dependable execution.

The design language should communicate:

- infrastructure-first thinking
- enterprise readiness
- API and platform orientation
- calm, modern SaaS clarity
- workflow and orchestration literacy
- recovery and operational trust
- developer confidence without developer chaos

The brand should not feel like a generic startup template, AI hype page, crypto interface, or playful consumer app.

## Core Identity

Primary identity:

- Navy: `#041B4D`
- Green: `#7ED321`
- White surfaces
- Soft mint sections
- Muted slate supporting text
- Thin-line workflow and infrastructure diagrams

The system should be clean enough for enterprise trust and flexible enough for developer tools, dashboards, docs, APIs, workflow diagrams, social cards, and product launches.

## Design North Star

Use this sentence to evaluate new work:

> Does this look like serious workflow infrastructure that can help teams operate authentication, recovery, payments, APIs, agents, and events with confidence?

If the answer is no, simplify the composition, reduce hype, increase structure, and return to the navy/green infrastructure language.

## Hard Rules

- Do not redesign the AuthToolkit logo in this repository.
- Keep navy and green as the primary identity.
- Use white, soft mint, and pale green as the dominant backgrounds.
- Keep diagrams thin-line, structured, and workflow-oriented.
- Prefer practical product screenshots, diagrams, and cards over abstract decoration.
- Do not use SaanaOS branding.
- Avoid neon, cyberpunk, crypto, gaming, and futuristic AI aesthetics.
- Avoid vague AI buzzwords when workflow, recovery, execution, and infrastructure language is clearer.

## Product Fit

This system must support:

- developer APIs
- operational dashboards
- authentication surfaces
- recovery and revenue workflows
- event and webhook orchestration
- agent execution infrastructure
- documentation pages
- landing pages
- social graphics
- admin portals

It should scale across products without each product inventing a new visual identity.
