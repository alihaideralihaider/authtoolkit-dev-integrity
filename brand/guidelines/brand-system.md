# Brand System

AuthToolkit is a calm infrastructure brand. It should communicate serious operational systems without becoming cold, generic, or overbuilt.

## Brand Attributes

Primary attributes:

- dependable
- structured
- API-first
- workflow-aware
- recovery-ready
- enterprise-capable
- developer-readable
- calm under pressure

Secondary attributes:

- modern
- clean
- precise
- execution-oriented
- scalable

Avoid attributes:

- magical
- futuristic
- playful
- chaotic
- consumer-social
- crypto-native
- gaming-inspired

## Visual Principles

### Structure Before Decoration

Every layout should have visible information architecture:

- clear primary action
- clear supporting action
- obvious hierarchy
- grouped evidence
- predictable spacing
- cards that represent real objects or decisions

Decoration should support workflow comprehension, not distract from it.

### Calm Infrastructure

Use whitespace, stable grids, soft mint backgrounds, and navy typography. Let green highlight the next action, active route, passing state, or safe pathway.

The brand should feel like a tool used during release review, recovery, payment operations, or authentication debugging.

### Operational Proof

Prefer concrete artifacts:

- API request cards
- event timelines
- workflow diagrams
- recovery paths
- dashboard panels
- release gates
- audit trails
- webhook routes

Avoid generic product clouds, floating abstract shapes, or decorative SaaS blobs.

## Color Identity

The identity is navy plus green, supported by soft mint, pale green, light gray, muted slate, and white.

Primary:

- Navy: `#041B4D`
- Green: `#7ED321`

Use navy for:

- typography
- headers
- footer bands
- diagram labels
- trust anchors
- deep background sections

Use green for:

- primary CTAs
- success states
- active paths
- positive workflow steps
- safe recovery indicators

Do not let green dominate full pages. It should signal action and health.

## Typography Identity

Approved families:

- Inter
- Poppins

Use Inter for product UI, docs, tables, dashboards, and dense operational content.

Use Poppins for marketing headlines, landing page section headings, and high-level product moments when a softer SaaS expression is useful.

If only one font can be used, use Inter.

## Messaging Identity

AuthToolkit messaging should explain operational outcomes.

Preferred themes:

- workflows
- recovery
- execution
- orchestration
- automation
- infrastructure
- approvals
- evidence
- operational simplicity
- confidence

Avoid overusing:

- AI
- autonomous
- AGI
- intelligent
- revolutionary
- magic
- effortless

Use AI language only when it clarifies the risk or workflow. Example:

Good:

> Reviewer confidence for AI-generated PRs.

Avoid:

> Revolutionary AI agents that magically ship code.

## Layout Personality

Layouts should be:

- spacious
- card-based when cards represent real concepts
- grid-driven
- mobile-first
- workflow-centric
- easy to scan

Avoid:

- dense walls of metrics
- giant decorative hero graphics with no meaning
- low-contrast gray text
- overlapping elements
- single-hue pages
- floating cards inside cards

## Visual References

The brand direction may take inspiration from the clarity of Stripe, Clerk, Retool, Twilio, Linear, and Rentros-style calm SaaS layouts, but AuthToolkit must remain infrastructure-first and operational.

Translate inspiration into AuthToolkit language:

- Stripe: API clarity and composable diagrams
- Clerk: auth infrastructure simplicity
- Retool: operational workflow density
- Twilio: communications infrastructure trust
- Linear: restrained product polish
- Rentros: soft gradients, whitespace, calm page rhythm

Do not copy any of these brands directly.

## System Scalability

Future products should share:

- palette
- typography
- button language
- diagram style
- cards and panels
- workflow visualization style
- docs tone
- social template structure

Product-specific pages may vary content and density, but should not create independent identities.
