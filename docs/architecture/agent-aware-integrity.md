# Agent-Aware Integrity

## Purpose

Agent-Aware Integrity adds AI-agent and automation authorship awareness to AuthToolkit Dev Integrity.

It detects local deterministic authorship signals and adjusts review expectations when agent or automation indicators intersect with sensitive engineering changes.

## Future of AI-Assisted Engineering

AI-assisted engineering can increase speed, but it changes the evidence burden.

When an agent or automation system contributes to sensitive code, the integrity system should require stronger human review, clearer evidence, and explicit approval boundaries.

Agent-Aware Integrity treats authorship as a signal, not proof.

## Authorship Signal Philosophy

v1 looks for simple deterministic indicators in changed file paths and safe diff-aware summaries:

- codex
- copilot
- cursor
- claude
- openai
- agent
- generated
- auto-generated
- ai-generated
- co-authored-by
- automated

It also looks for automation indicators:

- github-actions
- workflow
- bot
- script
- generated report
- generated file
- lockfile/package changes without source context

## Agent Risk Posture

Agent risk posture values:

- `human-likely`
- `agent-assisted`
- `agent-generated-suspected`
- `automation-generated`
- `unknown-authorship`

v1 does not prove who authored the change. It only records whether local signals suggest agent or automation involvement.

## Agent Trust Boundaries

Agent or automation signals become more important when they intersect with:

- critical architecture blast radius
- high or critical diff-aware findings
- payment, security, runtime, or Vault review packs
- release caution or block
- missing evidence
- policy escalation

## Review Requirements

Agent-Aware Integrity can require:

- independent human review
- security review for agent-touched auth/API changes
- payment review for agent-touched payment/webhook changes
- runtime review for agent-touched runtime/config changes
- architecture review for critical blast radius
- evidence attachment before approval

## Trust Warnings

Warnings include:

- Agent/automation signal detected near sensitive systems.
- Generated change requires independent review.
- Agent-touched change cannot rely only on generated report evidence.
- Policy escalation intersects with agent/automation signal.
- Critical blast radius requires human review even if generated by agent.

## Limitations

Agent-Aware Integrity v1 is deterministic and local-only.

It does not:

- identify a human author
- prove AI generated a change
- assign blame
- monitor employees
- enforce identity
- read GitHub approval metadata
- integrate with Jira

## What This Does Not Do Yet

Agent-Aware Integrity is not:

- AI detection certainty
- blame assignment
- employee monitoring
- identity enforcement
- GitHub/Jira integration
- approval automation
- SaaS analytics
