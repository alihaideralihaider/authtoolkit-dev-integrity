# Local Dashboard

Local Dashboard v1 is a read-only artifact viewer for AuthToolkit Dev Integrity.

## Purpose

The dashboard visualizes current operational trust state from generated local artifacts. It helps humans inspect recent reports, Control Room status, release gate score, active workflows, and timeline summary without opening every markdown file manually.

## Local Read-Only Artifact Viewer

The dashboard is plain HTML, CSS, and JavaScript under `tools/dev-integrity-dashboard/`. It has no build step and no framework.

It is served locally with:

```bash
npm run dashboard
```

## Data Sources

The dashboard reads:

- `reports/catalog.json`
- `reports/timeline-summary.md`
- latest report markdown linked from the catalog

These are generated by the existing review runner.

## No New Logic

The dashboard does not duplicate engine logic. It does not recalculate risk, release gate scores, workflow priority, Control Room status, or timelines. It only displays existing fields from generated artifacts.

## Safety

The dashboard does not:

- call external APIs
- write to GitHub
- deploy
- modify target repositories
- store secrets
- read raw logs
- use a database

## Future Dashboard Path

Future versions may add richer navigation, report detail drawers, charts over report history, and hosted Control Room surfaces. Those should still treat the Integrity Engine as the source of truth.

## Limitations

- v1 requires local generated artifacts to exist.
- v1 uses browser `fetch()`, so it should be served over a local static server.
- v1 does not parse full markdown reports beyond linking to them.
- v1 has no authentication because it is local-only.
