# New Project Onboarding

## Purpose

Register a new project with AuthToolkit Dev Integrity without importing app code or secret values.

## Steps

1. Create a project config with name, stack, repo path, and enabled review packs.
2. Create an env inventory with secret names only.
3. Identify critical routes, providers, deploy targets, and runtime bindings.
4. Select initial skills.
5. Run the first manual integrity review.
6. Store evidence locally or in the approved evidence store.

## Required Outputs

- project config
- env inventory
- first integrity report
- Vault readiness notes
- open questions
