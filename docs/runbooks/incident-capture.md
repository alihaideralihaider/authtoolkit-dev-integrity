# Incident Capture

## Purpose

Capture a production, deployment, security, or recovery incident in a structured way.

## Incident Record

- Incident title
- Severity
- Started at
- Detected by
- Affected project
- Affected environment
- User impact
- Root cause hypothesis
- Actions taken
- Evidence links
- Remaining risk
- Follow-up tasks

## Rules

- Do not include secret values.
- Do not include private customer records.
- Prefer evidence links and summaries over raw logs.
